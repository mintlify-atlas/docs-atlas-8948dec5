Esta guía te lleva desde cero hasta tener GIO TECH corriendo en tu máquina local y listo para ser desplegado en Firebase Hosting.

## Requisitos previos

Antes de empezar, asegúrate de tener instalado:

- **Node.js 18+** — [nodejs.org](https://nodejs.org)
- **npm** — incluido con Node.js
- **Firebase CLI** — para el despliegue en producción

Verifica tu versión de Node.js:

```bash
node --version
```

## Configuración paso a paso

<Steps>
  <Step title="Clona el repositorio e instala dependencias">
    Clona el repositorio de GIO TECH e instala todas las dependencias del proyecto.

    <CodeGroup>

    ```bash npm
    git clone https://github.com/Est3banj/gio-tech.git
    cd gio-tech
    npm install
    ```

    ```bash yarn
    git clone https://github.com/Est3banj/gio-tech.git
    cd gio-tech
    yarn install
    ```

    </CodeGroup>
  </Step>

  <Step title="Crea un proyecto en Firebase">
    GIO TECH usa tres servicios de Firebase. Actívalos en la consola de Firebase:

    1. Ve a [console.firebase.google.com](https://console.firebase.google.com) y crea un nuevo proyecto.
    2. Activa **Authentication** → habilita el proveedor **Correo electrónico/Contraseña**.
    3. Activa **Firestore Database** → elige el modo **producción** y selecciona la región más cercana.
    4. Activa **Storage** → acepta las reglas predeterminadas (puedes ajustarlas después).
    5. En **Configuración del proyecto → Tus aplicaciones**, registra una app web y copia las credenciales.
  </Step>

  <Step title="Configura las variables de entorno">
    Crea el archivo `.env` en la raíz del proyecto con las credenciales de tu proyecto Firebase:

    ```bash .env
    VITE_FIREBASE_API_KEY=tu_api_key
    VITE_FIREBASE_AUTH_DOMAIN=tu_proyecto.firebaseapp.com
    VITE_FIREBASE_PROJECT_ID=tu_proyecto
    VITE_FIREBASE_STORAGE_BUCKET=tu_proyecto.appspot.com
    VITE_FIREBASE_MESSAGING_SENDER_ID=123456789012
    VITE_FIREBASE_APP_ID=1:123456789012:web:abc123def456
    ```

    Encuentra estos valores en la consola de Firebase: **Configuración del proyecto → General → Tus aplicaciones → SDK setup and configuration**.

    <Warning>
      No subas el archivo `.env` al repositorio. El proyecto ya incluye `.env` en `.gitignore`. Nunca expongas estas credenciales en código público.
    </Warning>
  </Step>

  <Step title="Levanta el servidor de desarrollo">
    Inicia el servidor de desarrollo local con Vite:

    ```bash
    npm run dev
    ```

    La aplicación estará disponible en `http://localhost:5173`. Los cambios se reflejan en tiempo real sin necesidad de recargar la página.
  </Step>

  <Step title="Accede al panel de administración">
    Para ingresar al panel de administración necesitas un usuario con rol `admin` en Firestore.

    1. Navega a `/login` en tu navegador local.
    2. Inicia sesión con las credenciales del usuario admin.
    3. Serás redirigido automáticamente a `/panel` → **AdminPanel**.

    <Tip>
      Para crear el primer usuario administrador, ve a la consola de Firebase → **Authentication** y crea un usuario con correo y contraseña. Luego, en **Firestore Database**, crea manualmente el documento en la ruta `usuarios/{uid}` con el campo `rol: "admin"` y `nombreCompleto: "Tu nombre"`. El `uid` lo encuentras en la pestaña Authentication junto al correo del usuario.
    </Tip>
  </Step>
</Steps>

## Scripts disponibles

Una vez instalado el proyecto, tienes los siguientes comandos de npm:

| Comando | Descripción |
|---|---|
| `npm run dev` | Inicia el servidor de desarrollo local en `http://localhost:5173` |
| `npm run build` | Genera el build de producción optimizado en la carpeta `dist/` |
| `npm run preview` | Previsualiza el build de producción localmente antes de desplegar |
| `npm run deploy` | Construye el proyecto y lo despliega en Firebase Hosting |
| `npm run lint` | Ejecuta ESLint para verificar la calidad del código |

## Desplegar en producción

Para publicar GIO TECH en Firebase Hosting, instala el CLI de Firebase y ejecuta el despliegue:

```bash
npm install -g firebase-tools
firebase login
npm run deploy
```

El comando `npm run deploy` ejecuta `npm run build && firebase deploy` internamente, generando el build en `dist/` y desplegándolo al proyecto `gio-tech` configurado en `.firebaserc`.

El sitio quedará disponible en `https://gio-tech.web.app` (o el dominio personalizado que hayas configurado en Firebase Hosting).
