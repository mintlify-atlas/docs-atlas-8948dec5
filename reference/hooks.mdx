## useProducts

Suscribe al componente a la colecciĂłn `productos` de Firestore en tiempo real. Gracias al store externo compartido, todos los componentes que llamen a `useProducts` reciben el mismo array sin abrir mĂşltiples listeners.

### Firma

```javascript
useProducts() â†’ { products: object[], isLoading: boolean, error: Error | null }
```

### Valor de retorno

<ResponseField name="products" type="object[]">
  Array de productos. Cada objeto incluye el `id` del documento de Firestore mĂˇs todos sus campos (`nombre`, `contado`, `cuotas6`, `cuotas8`, `imagen`, `marca`, etc.).
</ResponseField>

<ResponseField name="isLoading" type="boolean">
  `true` mientras se espera la primera respuesta de Firestore. Pasa a `false` en cuanto llega el primer snapshot (con o sin productos).
</ResponseField>

<ResponseField name="error" type="Error | null">
  `null` en condiciones normales. Contiene el error si el listener de Firestore falla.
</ResponseField>

### Ejemplo de uso

```jsx Catalogo.jsx
import { useProducts } from '../hooks';

function Catalogo() {
  const { products, isLoading, error } = useProducts();

  if (isLoading) return <p>Cargando productos...</p>;
  if (error) return <p>Error al cargar productos.</p>;

  return (
    <ul>
      {products.map((p) => (
        <li key={p.id}>
          {p.nombre} â€” ${p.contado?.toLocaleString('es-CO')}
        </li>
      ))}
    </ul>
  );
}
```

<Tip>
  Si ya existe data en el store externo cuando un nuevo componente monta, `isLoading` se establece en `false` de inmediato y no se abre una segunda suscripciĂłn a Firestore.
</Tip>

## useConfig

Suscribe al componente al documento `configuracion/general` de Firestore en tiempo real. Comparte el mismo patrĂłn singleton que `useProducts`.

### Firma

```javascript
useConfig() â†’ { config: object, isLoading: boolean, error: Error | null }
```

### Valor de retorno

<ResponseField name="config" type="object">
  Objeto con todos los campos del documento `configuracion/general`. Si el documento no existe en Firestore, devuelve `{}`.
</ResponseField>

<ResponseField name="isLoading" type="boolean">
  `true` durante la carga inicial.
</ResponseField>

<ResponseField name="error" type="Error | null">
  `null` en condiciones normales.
</ResponseField>

### Ejemplo de uso

```jsx Header.jsx
import { useConfig } from '../hooks';

function Header() {
  const { config, isLoading } = useConfig();

  if (isLoading) return null;

  return (
    <header>
      {config.logo && <img src={config.logo} alt={config.nombre ?? 'GIO TECH'} />}
      <h1>{config.nombre ?? 'GIO TECH'}</h1>
    </header>
  );
}
```

## useAuth

Combina `onAuthStateChanged` de Firebase Auth con un listener de Firestore sobre `usuarios/{uid}` para enriquecer el objeto de usuario con su rol y nombre completo. Gestiona la limpieza de ambas suscripciones automĂˇticamente.

### Firma


### Valor de retorno

<ResponseField name="user" type="object | null">
  `null` si no hay sesiĂłn activa. Cuando el usuario estĂˇ autenticado, el objeto tiene la siguiente forma:

  <Expandable title="Propiedades del objeto user">
    <ResponseField name="uid" type="string">
      UID de Firebase Auth.
    </ResponseField>
    <ResponseField name="email" type="string">
      Correo electrĂłnico del usuario.
    </ResponseField>
    <ResponseField name="rol" type="string">
      Rol del usuario en la plataforma: `"admin"`, `"asesor"` o `"cliente"`. Mientras se carga desde Firestore muestra `"cargando..."`. Si el documento no existe, se asigna `"cliente"`.
    </ResponseField>
    <ResponseField name="nombreCompleto" type="string">
      Nombre completo leĂ­do del documento `usuarios/{uid}`. Puede ser `undefined` si el campo no existe.
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="isLoading" type="boolean">
  `true` mientras Firebase Auth resuelve el estado inicial. Pasa a `false` tras obtener (o descartar) el usuario y sus datos de Firestore.
</ResponseField>

### Ejemplo de uso

```jsx ProtectedRoute.jsx

  const { user, isLoading } = useAuth();

  if (isLoading) return <p>Verificando sesiĂłn...</p>;
  if (!user) return <Navigate to="/login" />;
  if (!rolesPermitidos.includes(user.rol)) return <Navigate to="/" />;

  return children;
}
```

```jsx NavBar.jsx

  const { user } = useAuth();

  return (
    <nav>
      {user ? (
        <span>Bienvenido, {user.nombreCompleto} ({user.rol})</span>
      ) : (
        <a href="/login">Iniciar sesiĂłn</a>
      )}
```

## usePopularProducts

Obtiene los IDs de los productos mĂˇs vistos consultando la colecciĂłn `producto_stats` ordenada por `vistas` descendente. Utiliza `getPopularProductsStats` del servicio de estadĂ­sticas.

```jsx ProductosDestacados.jsx
  const { popularIds, isLoading: loadingIds } = usePopularProducts();
  const { products, isLoading: loadingProducts } = useProducts();

    <section>
      <h2>MĂˇs vistos</h2>
      {destacados.map((p) => (
        <article key={p.id}>
          <img src={p.imagen} alt={p.nombre} />
          <p>{p.nombre}</p>
        </article>
      ))}
    </section>
  );
}
```
